### **Sign Up Experience**

The Sign-Up screen will be one of the first interactions with Anagram, making it a crucial part of the overall user experience. The screen is designed to minimise effort and hesitation, allowing students to create an account quickly and begin organising their study without distraction.

![[Sign-up story board.png]]

The layout of the Sign-Up page is fully responsive, ensuring consistent behaviour across mobile and desktop devices. From the application’s landing page, users are clearly directed to account creation, reducing friction during onboarding.

The interface centers around a concise registration form that collects essential information such as full name, email address, password, and year level. Input fields are clearly labelled and supported by placeholder text to guide users through the process. Validation occurs as data is entered, providing dynamic feedback for issues such as weak passwords or invalid email formats. Password input is masked to maintain privacy.

Security considerations are embedded directly into the experience. Input validation prevents incomplete or malformed data from being submitted, while clear password requirements lead to stronger credentials. These measures support secure account creation without overwhelming the user.

Once registration is successful, users are automatically redirected to the main dashboard. This immediate transition allows students to begin adding subjects and planning study sessions without unnecessary intermediate steps, improving the fluidity of the user experience.

The visual design remains intentionally simple. Consistent typography, spacing, and colour usage maintains a consistent theme that guides attention and reduces cognitive load. This ensures the Sign Up process feels approachable and aligned with Anagram’s goal of supporting structured and stress free study planning.
### **Task Creation

Creating a new study task is one of the main functions within Anagram, as it supports students in breaking down their workload into manageable actions. The task creation flow is designed to be fast, guided, and low effort, encouraging consistent use rather than avoidance.
![[Manual add task story board.png]]
From the dashboard, users can initiate the process by selecting the **“Create new task”** action. This immediately signals a focused workflow, separating task creation from broader planning views and reducing distractions.

The task input screen presents a structured form that collects essential details including task title, task type, date, and time. Dropdown selectors and input constraints are used to minimise errors, reducing confusion, and support validation. Clear labelling and spacing guide the user through the form in a logical top too bottom sequence.

As users enter information, the interface provides dynamic validation feedback to prevent incomplete or invalid submissions. This makes sure tasks are created with meaningful scheduling data, reducing the possibility of confusion later. The design prioritises clarity over density, making the process approachable for high stress periods.

Once a task is successfully added, the user is presented with a confirmation screen. This visual feedback, represented by a clear success indicator, reassures the user that their action has been completed correctly. A single **“Continue”** action allows users to return seamlessly to planning or task review without interrupting their workflow.

The overall visual style remains consistent with Anagram’s scheme. Rounded elements, restrained colour usage, and generous whitespace reduce cognitive load and support quick interactions. This experience reinforces Anagram’s goal of helping students organise study tasks efficiently without adding unnecessary friction or complexity.
